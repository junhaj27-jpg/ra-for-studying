## 🧠 도메인 확장 가능성: 의료 AI / Brain MRI 분석 시스템

Brain MRI 3D Longitudinal Analysis Web Service는 9년간의 장기 MRI 데이터와 1개월 단기 집중 촬영 데이터를 기반으로 병변 부피 변화, 3D 모델링, 시계열 추적 분석을 제공하는 웹 기반 진단 보조 프로토타입입니다.

ALPLED는 특정 산업에 한정된 플랫폼이 아니라, RFP와 회의록, 기획 자료를 기반으로 다양한 도메인의 개발 산출물을 자동 생성할 수 있는 AI-DLC 문서 생성 플랫폼입니다.

특히 의료 AI 분야는 요구사항 정의, 데이터 보안, 검증 문서, 테스트 시나리오 작성이 중요한 영역이기 때문에 ALPLED의 확장 적용 가능성이 높습니다.

### Brain MRI 분석 시스템 적용 예시

예를 들어 Brain MRI 기반 종양 부피 추적 시스템을 개발한다고 가정하면, ALPLED는 해당 프로젝트에 필요한 개발 산출물을 자동 생성할 수 있습니다.

### 보안 및 폐쇄망 적용 가능성

의료 데이터는 개인정보와 민감정보를 포함할 수 있기 때문에 외부 LLM API로 원본 데이터를 전송하기 어렵습니다.
ALPLED는 자체 sLLM 기반 구조를 지향하기 때문에 의료기관 내부망 또는 전용 GPU 서버 환경에서도 적용 가능한 구조를 목표로 합니다.

이를 통해 원본 의료 데이터와 RAG 문서는 외부 저장소에 업로드하지 않고, 내부 환경에서 문서 생성과 검증을 수행할 수 있습니다.

> # Brain MRI 3D Longitudinal Analysis Web Service

## 1. 프로젝트 개요

본 프로젝트는 동일 환자의 장기 Brain MRI 데이터를 기반으로 병변 또는 종양 의심 영역을 분석하고, 촬영일별 부피 변화를 추적하는 웹 기반 MRI 분석 서비스입니다.

사용자는 MRI 데이터를 업로드하고, 환자 코드·촬영일·병원명·촬영 장비 정보를 기준으로 데이터를 관리할 수 있습니다.
시스템은 MRI 슬라이스 데이터를 기반으로 2D 뷰어, 병변 부피 분석, 3D 모델 뷰어, 장기 추적 그래프, PDF 분석 보고서 기능을 제공합니다.

본 서비스는 의료진의 진단을 대체하는 목적이 아니라, Brain MRI 분석 및 장기 추적을 보조하기 위한 프로토타입입니다.

---

## 2. 개발 배경

Brain MRI는 장기간 추적 관찰이 필요한 질환에서 중요한 의료영상 데이터입니다.
특히 종양 또는 병변의 크기 변화는 환자의 상태 변화, 치료 반응, 재발 가능성 등을 확인하는 데 중요한 참고 자료가 됩니다.

본 프로젝트의 데이터는 다음과 같은 특징을 가집니다.

| 구분        | 내용                                  |
| --------- | ----------------------------------- |
| 장기 추적 데이터 | 서울대병원에서 약 9년간 촬영된 Brain MRI 데이터     |
| 단기 집중 데이터 | 이대목동병원에서 약 1개월 동안 촬영된 약 4개 MRI 세트   |
| MRI 규모    | MRI 1세트당 약 895개의 슬라이스 이미지           |
| 분석 가능성    | 3D 모델링 및 병변 부피 추적이 가능한 수준의 고해상도 데이터 |

서로 다른 병원과 촬영 장비에서 생성된 MRI 데이터는 밝기, 해상도, 슬라이스 간격, 방향 정보가 다를 수 있습니다.
따라서 본 서비스는 병원 및 장비 차이를 고려하여 MRI 데이터를 관리하고, 장기적인 부피 변화 추적을 지원하는 것을 목표로 합니다.

## 3. 주요 기능

## 3.1 MRI 데이터 관리

* 환자 코드 기반 MRI 데이터 관리
* 촬영일, 병원명, 장비명 정보 등록
* MRI 파일 세트 업로드
* MRI 데이터 목록 조회
* 분석 상태 확인
* MRI 원본 파일 및 결과 파일 다운로드

## 3.2 MRI 파일 업로드

* DICOM, NIfTI, ZIP 형식의 MRI 파일 업로드 지원
* MRI 1세트당 다수의 슬라이스 파일 관리
* 파일 형식, 용량, 개수 검증
* 업로드 진행 상태 표시
* 업로드 실패 시 오류 메시지 제공

## 3.3 MRI 전처리

* MRI 슬라이스 순서 정렬
* 밝기 및 명암 정규화
* 병원 및 장비 차이에 따른 해상도 보정
* 3D 분석을 위한 볼륨 데이터 구성
* 웹 표시용 PNG 슬라이스 변환

## 3.4 AI 기반 MRI 분석

* Brain MRI 내 병변 또는 종양 의심 영역 분석
* Segmentation mask 생성
* 원본 MRI와 mask overlay 표시
* 병변 부피 계산
* 분석 상태 관리

## 3.5 3D 모델링

* MRI 슬라이스 기반 3D 볼륨 구성
* Segmentation mask 기반 병변 3D 모델 생성
* Three.js 기반 웹 3D 뷰어 제공
* 3D 모델 회전, 확대, 축소 기능 제공
* 병변 또는 종양 영역의 공간적 위치 확인

## 3.6 장기 추적 분석

* 동일 환자 코드 기준 촬영 이력 조회
* 서울대병원 9년 장기 MRI 변화 추적
* 이대목동병원 1개월 단기 MRI 변화 비교
* 촬영일별 병변 부피 변화량 계산
* 병변 부피 변화율 계산
* 부피 변화 그래프 제공

## 3.7 분석 보고서

* MRI 분석 결과 PDF 보고서 생성
* 환자 코드, 촬영일, 병원명, 장비명 표시
* 병변 부피, 변화량, 변화율 표시
* 원본 이미지, mask 이미지, overlay 이미지 포함
* 진단 보조용 안내 문구 포함

## 8. 데이터베이스 설계 요약

| 테이블              | 설명                       |
| ---------------- | ------------------------ |
| User             | 사용자 계정 및 권한 정보           |
| Project          | MRI 분석 프로젝트 정보           |
| Patient          | 비식별 환자 코드 정보             |
| MRIStudy         | 촬영일, 병원명, 장비명 등 촬영 단위 정보 |
| MRISeries        | MRI 파일 세트 및 슬라이스 정보      |
| AnalysisJob      | MRI 분석 요청 및 진행 상태 정보     |
| AnalysisResult   | 분석 결과, 병변 부피, 신뢰도 정보     |
| SegmentationMask | 병변 mask 파일 정보            |
| Model3D          | 3D 모델 파일 정보              |
| Report           | PDF 분석 보고서 정보            |

## 9. 개발 단계

## 9.1 1차 MVP

* 로그인 기능
* 프로젝트 선택 기능
* 환자 코드 기반 MRI 데이터 관리
* MRI 파일 업로드
* 촬영일, 병원명, 장비명 등록
* MRI 목록 조회
* 분석 상태 표시

## 9.2 2차 MVP

* MRI 슬라이스 PNG 변환
* 웹 기반 MRI 슬라이스 뷰어
* 촬영일별 MRI 이력 조회
* 병원별 MRI 데이터 구분
* 분석 결과 목록 화면

## 9.3 3차 MVP

* 병변 mask 등록 또는 생성
* 병변 부피 계산
* 촬영일별 부피 변화 그래프
* 단기/장기 추적 분석
* PDF 보고서 생성

## 9.4 4차 MVP

* 3D 모델 생성
* Three.js 기반 3D 모델 뷰어
* 병원 및 장비 차이 보정 결과 표시
* AI segmentation 모델 연결

## 10. AI 활용 개발 기간 예상

AI를 최대한 활용하여 개발할 경우 예상 기간은 다음과 같습니다.

| 개발 수준              |  예상 기간 |
| ------------------ | -----: |
| 문서 산출물 작성          |     1일 |
| 화면 데모 제작           |   2~3일 |
| 업로드 및 목록 관리 MVP    | 1~1.5주 |
| MRI 슬라이스 뷰어 포함 MVP |   2~3주 |
| 부피 계산 포함 MVP       |   3~4주 |
| 3D 모델 뷰어 포함 MVP    |   4~5주 |
| AI 자동 분석 포함 MVP    |   6~8주 |

본 프로젝트는 AI를 활용하여 요구사항정의서, 화면설계서, DB 설계서, 테스트 시나리오, 코드 초안을 빠르게 생성할 수 있습니다.
다만 MRI 데이터 정리, 병원별 촬영 조건 차이 확인, 3D 모델 품질 검토, AI 분석 결과 검증은 사람이 직접 확인해야 합니다.

## 11. 기대 효과

* 장기 Brain MRI 데이터의 체계적 관리
* 병원 및 장비가 다른 MRI 데이터의 비교 가능성 확보
* 병변 또는 종양 의심 영역의 부피 정량화
* 촬영일별 변화량 및 변화율 확인
* 2D 슬라이스와 3D 모델을 활용한 시각적 분석
* 의료진 및 연구자를 위한 진단 보조 자료 제공
* 포트폴리오용 의료 AI 웹서비스 구현 사례 확보

---

## 12. 주의사항

본 프로젝트는 연구 및 포트폴리오 목적의 MRI 분석 웹서비스 프로토타입입니다.
본 시스템의 분석 결과는 의료진의 최종 진단을 대체할 수 없으며, 실제 임상 환경에서 사용하기 위해서는 추가적인 검증, 보안 심사, 의료기기 인허가 절차가 필요합니다.

분석 결과 화면 및 보고서에는 다음 문구를 포함합니다.

> 본 분석 결과는 의료진의 진단을 보조하기 위한 참고 자료이며, 최종 진단 및 치료 판단은 의료 전문가가 수행해야 합니다.
