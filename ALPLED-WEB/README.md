# AIDLC-RA WEB Console

AIDLC-RA WEB은 `AIDLC-RA: AI-DLC Based Regulatory Affairs Documentation Platform`의 Django 기반 웹 콘솔입니다. 기존 ALPLED의 산출물 생성, 버전 관리, 승인 요청, OnlyOffice 편집 흐름은 유지하면서 화면 문구와 문서 흐름을 의료기기 RA 취업 포트폴리오 관점으로 재구성했습니다.

> 본 프로젝트는 의료 진단 또는 치료 목적의 소프트웨어가 아니라, 의료기기 개발 및 RA 문서 작성 과정을 보조하기 위한 포트폴리오용 문서 자동화 플랫폼입니다.

## 웹 콘솔의 역할

AIDLC-RA WEB은 의료기기 실제 인허가를 대신하지 않습니다. 화면의 목표는 RA 담당자 또는 개발자가 개발 자료를 업로드하고, 요구사항, 위험관리, 검증시험, 변경관리, 추적성 매트릭스를 일관된 문서 흐름으로 관리하는 모습을 보여주는 것입니다.

## 주요 화면

| 메뉴 | 화면 목적 |
| --- | --- |
| Dashboard | RA 프로젝트별 문서화 진행 상태와 누락 문서 현황 확인 |
| Project Upload | 제품 개요, 회의록, 요구사항, 시험 자료, 변경 이력 업로드 |
| RA Document Generator | RAG와 템플릿 기반 RA 산출 문서 초안 생성 |
| Risk Management | 위해요인, 위해상황, 위험통제, 검증 항목 연결 |
| Traceability Matrix | 요구사항, 위험, 시험, 검증 결과, 변경 이력 연결 |
| Test Scenario | 요구사항과 위험통제 기반 통합시험 시나리오 생성 |
| Validation Report | RA 문서 정합성 검토 결과, 누락, 불일치, 미검증 항목 확인 |
| Admin | 사용자, 프로젝트, 규제 템플릿, 출력 형식 관리 |

## 문서 유형 매핑

기존 문서 코드와 라우팅은 유지하고, 화면 라벨과 설명을 RA 문서 체계로 재정의했습니다.

| 기존 코드 | RA 문서명 | 목적 |
| --- | --- | --- |
| DOC_SRS | 요구사항정의서 | 사용 목적, 사용자, 기능/비기능, 규제 요구사항 정의 |
| DOC_ITF | 위험관리표 | 위해요인, 위해상황, 위험통제, 검증항목 연결 |
| DOC_ARCH | 소프트웨어 요구사항 명세서 | SaMD 개발문서 보조를 위한 SRS 작성 |
| DOC_ERD | 추적성 매트릭스 | 요구사항, 위험, 시험, 검증 결과 연결 |
| DOC_DB | 변경관리 기록 | 변경 영향, 재검증 필요 여부, 인허가 영향 기록 |
| DOC_TS | 통합시험 시나리오 | 요구사항과 위험통제 기반 시험 절차 생성 |

## RA 관점 강화 사항

- “문서 생성”을 “RA 문서 생성”으로 재정의
- “프로젝트 업로드”를 “의료기기 개발 자료 업로드”로 변경
- 승인/검증 흐름을 “RA 문서 정합성 검토 결과”로 표현
- 진단, 처방, 치료 판단 기능이 없다는 범위 고지 추가
- 결과 상세 화면에 요구사항, 위험관리, 시험 시나리오, 변경관리 연결 구조 추가
- 생성 에이전트와 검증 에이전트가 분리된 포트폴리오 구조를 README에 설명
