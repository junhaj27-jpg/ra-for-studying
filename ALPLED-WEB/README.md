# AIDLC WEB

AIDLC WEB은 AI-DLC 기반 의료기기 RA 문서 자동화 플랫폼의 Django 웹 콘솔입니다. 기존 ALPLED의 산출물 생성, 버전 관리, 승인 요청, OnlyOffice 편집 흐름을 유지하면서 화면과 문서 체계를 의료기기 RA/인허가 문서 작성 보조 흐름으로 재구성했습니다.

> 본 프로젝트는 의료 진단 또는 치료 목적의 소프트웨어가 아니라, 의료기기 개발 및 RA 문서 작성 과정을 보조하기 위한 포트폴리오용 문서 자동화 플랫폼입니다.

## 주요 화면

- Dashboard: RA 프로젝트와 문서 진행 상태 확인
- Project Upload: 의료기기 개발 자료 업로드
- RA Document Generator: RA 문서 생성
- Risk Management: 위험관리표 작성 및 위험-검증 연결
- Traceability Matrix: 요구사항-위험-시험-검증 결과 추적
- Test Scenario: 통합시험 시나리오 생성
- Validation Report: RA 문서 정합성 검토 결과 확인
- Admin: 사용자, 프로젝트, 규제 템플릿 관리

## 문서 유형 매핑

기존 문서 코드와 라우팅은 유지하고, 화면 라벨과 설명을 RA 문서 체계로 재정의했습니다.

| 기존 코드 | RA 문서명 | 목적 |
| --- | --- | --- |
| DOC_SRS | 요구사항정의서 | 사용 목적, 사용자, 기능/비기능, 규제 요구사항 정의 |
| DOC_ITF | 위험관리표 | 위해요인, 위해상황, 위험통제, 검증항목 연결 |
| DOC_ARCH | 소프트웨어 요구사항 명세서 | SaMD 개발문서 보조를 위한 SRS 작성 |
| DOC_ERD | 추적성 매트릭스 | 요구사항, 위험, 시험, 검증 결과 연결 |
| DOC_DB | 변경관리 기록 | 변경 영향, 재검증 필요 여부, 인허가 영향 기록 |
| DOC_TS | 통합시험 시나리오 | 요구사항과 위험통제 기반 시험 절차 생성 |

## RA 관점 강화 사항

- 문서 생성을 RA 문서 생성으로 재정의
- 프로젝트 업로드를 의료기기 개발 자료 업로드로 변경
- 승인/검증 흐름을 RA 문서 정합성 검토 흐름으로 표현
- 진단, 처방, 치료 판단 기능이 없다는 범위 고지 추가
- 문서 간 추적성과 누락/불일치 검토를 포트폴리오 핵심 가치로 명시
