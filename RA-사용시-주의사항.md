# RA 사용 시 주의사항

이 문서는 AIDLC-RA 프로젝트를 사용할 때 가장 먼저 확인해야 하는 RA 관점 주의사항입니다.

> 본 프로젝트는 의료 진단 또는 치료 목적의 소프트웨어가 아니라, 의료기기 개발 및 RA 문서 작성 과정을 보조하기 위한 포트폴리오용 문서 자동화 플랫폼입니다.

## 먼저 확인할 것

- 생성 결과는 초안이며 실제 인허가 제출 문서나 품질 승인 문서를 대체하지 않습니다.
- 의료 진단, 치료 결정, 처방 추천, 응급상황 판단, 임상적 유효성 판단에 사용하지 않습니다.
- 최신 법령, 고시, 국제표준, 기관별 양식, 품질시스템 절차와 반드시 대조해야 합니다.
- ISO 14971 위험도 계산과 IEC 62304 추적성 결과는 RA 검토를 돕는 보조 지표입니다.
- RAG 근거 문장과 출처는 원문, 버전, 작성일, 적용 범위를 직접 확인해야 합니다.
- 환자정보, 개인정보, 영업비밀, 승인되지 않은 임상자료는 샘플이나 근거 문서에 넣지 않습니다.

## RA 검토 체크리스트 요약

- 사용 목적이 진단, 치료, 처방, 응급 판단으로 오해되지 않는가
- 요구사항 ID, 위험 ID, 시험 ID, 변경 ID가 일관되는가
- 위험통제 항목이 검증시험 또는 근거 문서와 연결되는가
- 변경사항이 SRS, 위험관리표, 시험 시나리오, 추적성 매트릭스에 반영되는가
- Word/PDF 출력 전 문서번호, 버전, 승인권자, 개정이력, 서명란을 확인했는가

상세 기준은 `docs/ra-usage-precautions.md`를 확인하세요.
