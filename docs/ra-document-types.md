# AIDLC-RA 문서 유형 정의

이 문서는 AIDLC-RA 플랫폼에서 다루는 RA 문서 유형과 각 문서의 입력값, 출력값, 검토 기준을 정의합니다. 목적은 의료기기 인허가 문서를 대신 작성하는 것이 아니라, RA 실무 흐름을 이해한 포트폴리오형 문서 자동화 구조를 설명하는 것입니다.

> 본 프로젝트는 의료 진단 또는 치료 목적의 소프트웨어가 아니라, 의료기기 개발 및 RA 문서 작성 과정을 보조하기 위한 포트폴리오용 문서 자동화 플랫폼입니다.

## 공통 검토 원칙

- 사용 목적과 문서 범위가 일치해야 합니다.
- 진단, 치료, 처방, 응급 판단으로 오해될 표현을 사용하지 않습니다.
- 요구사항 ID, 위험 ID, 시험 ID, 변경 ID가 추적 가능해야 합니다.
- 위험통제 항목은 검증 및 밸리데이션 계획서 또는 통합시험 시나리오에 연결되어야 합니다.
- 변경사항은 영향받는 요구사항, 위험관리표, 시험 시나리오, 추적성 매트릭스에 반영되어야 합니다.

## 1. 요구사항정의서

목적: 제품의 사용 목적, 사용자, 기능, 제약사항, 규제 고려사항을 초기 개발 단계에서 정리합니다.

필수 항목:

- 제품 개요
- 사용 목적
- 대상 사용자
- 사용 환경
- 기능 요구사항
- 비기능 요구사항
- 규제 요구사항
- 제외 범위
- 관련 근거 문서

검토 기준:

- 의료적 주장과 제품 범위가 일치하는가
- 진단, 처방, 치료 판단 기능으로 오해될 표현이 없는가
- 이후 SRS와 추적 가능한 요구사항 ID가 있는가
- 위험관리표로 연결해야 할 안전 관련 요구사항이 식별되었는가

## 2. 소프트웨어 요구사항 명세서

목적: SaMD 또는 의료기기 관련 소프트웨어의 기능 요구사항을 검증 가능한 형태로 정리합니다.

필수 항목:

- SRS ID
- 기능 설명
- 입력 데이터
- 출력 데이터
- 예외 처리
- 성능 기준
- 보안 요구사항
- 관련 위험 ID
- 관련 시험 ID

검토 기준:

- 각 요구사항이 테스트 가능한 문장인가
- 위험관리표 및 시험 시나리오와 연결되는가
- 사용자에게 의료 판단으로 오해될 수 있는 결과 표현이 없는가
- 요구사항 변경 시 변경관리 기록과 추적성 매트릭스가 갱신되는가

## 3. 위험관리표

목적: ISO 14971 관점의 위해요인, 위해상황, 위해, 위험통제, 잔여위험을 구조화합니다.

필수 항목:

- 위험 ID
- 위해요인
- 위해상황
- 위해
- 발생 가능성
- 심각도
- 초기 위험
- 위험통제 방법
- 검증 항목
- 잔여위험
- 허용 여부

검토 기준:

- 위험통제 방법이 검증 항목과 연결되는가
- 잔여위험이 허용 가능하다는 근거가 있는가
- 사용설명서 또는 주의사항 반영이 필요한가
- 요구사항 또는 변경관리 기록과 연결되는가

## 4. 검증 및 밸리데이션 계획서

목적: 제품 요구사항과 소프트웨어 요구사항이 적절히 검증되는지 계획합니다.

필수 항목:

- 검증 범위
- 검증 대상 요구사항
- 시험 환경
- 시험 데이터
- 합격 기준
- 책임자
- 일정
- 산출물

검토 기준:

- 요구사항별 검증 방법이 누락되지 않았는가
- 위험통제 항목이 시험으로 확인되는가
- 성능 주장과 합격 기준이 일치하는가
- 시험 결과가 추적성 매트릭스에 반영될 수 있는 구조인가

## 5. 통합시험 시나리오

목적: 실제 사용자 흐름과 시스템 기능을 기반으로 통합시험 케이스를 구성합니다.

필수 항목:

- 시험 ID
- 관련 요구사항 ID
- 관련 위험 ID
- 사전 조건
- 시험 절차
- 기대 결과
- 합격 기준
- 실제 결과
- 판정

검토 기준:

- 요구사항과 시험 절차가 매핑되는가
- 위험관리표의 통제 수단이 시험에 반영되는가
- 오류 또는 예외 상황 테스트가 포함되는가
- 시험 결과가 검증 및 밸리데이션 보고서로 이어질 수 있는가

## 6. 변경관리 기록

목적: 요구사항, 알고리즘, 데이터, UI, 라벨, 사용설명서 변경이 RA 문서와 검증 자료에 미치는 영향을 기록합니다.

필수 항목:

- 변경 ID
- 변경 유형
- 변경 사유
- 영향받는 요구사항
- 영향받는 위험 항목
- 재검증 필요 여부
- 인허가 영향
- 승인자
- 승인일

검토 기준:

- 변경에 따른 문서 업데이트가 누락되지 않았는가
- 재검증 필요 여부가 판단되었는가
- 변경 후 추적성 매트릭스가 유지되는가
- 사용 목적 또는 의료적 주장에 영향을 주는 변경인지 검토되었는가

## 7. 추적성 매트릭스

목적: 요구사항, 위험, 시험, 검증 결과, 변경 이력을 하나의 표로 연결합니다.

필수 항목:

- 요구사항 ID
- 위험 ID
- 시험 ID
- 검증 결과 ID
- 변경 ID
- 상태
- 비고

검토 기준:

- 요구사항 중 시험과 연결되지 않은 항목이 없는가
- 위험통제 중 검증되지 않은 항목이 없는가
- 변경 후에도 추적성이 유지되는가
- 문서 간 ID와 용어가 일관되는가

## 결과 화면에서 보여줄 연결 구조

Validation Report 화면은 단순 승인 여부가 아니라 다음 구조를 설명해야 합니다.

| 연결 | 검토 질문 |
| --- | --- |
| 요구사항 -> 위험관리 | 안전 관련 요구사항이 위험관리표에 반영되었는가 |
| 위험통제 -> 검증시험 | 위험통제 방법이 시험 시나리오로 확인되는가 |
| 시험 -> 검증 결과 | 합격 기준과 실제 결과가 기록되는가 |
| 변경관리 -> 재검증 | 변경으로 인해 재검증이 필요한 항목이 식별되었는가 |
| 전체 문서 -> 추적성 매트릭스 | 요구사항, 위험, 시험, 결과, 변경이 하나의 ID 체계로 연결되는가 |
## RA 사용 주의사항 및 검토 기준

AIDLC-RA의 산출 문서는 실제 제출 문서가 아니라 RA 문서 작성 보조 결과입니다. 각 문서 유형별 검토 기준을 적용하기 전에 다음 공통 주의사항을 확인해야 합니다.

- 생성 결과를 최종 인허가 문서 또는 품질 승인 문서로 바로 사용하지 않습니다.
- 의료 진단, 치료 결정, 처방 추천, 응급상황 판단에 사용하지 않습니다.
- 최신 법령, 고시, 국제표준, 기관별 양식, 품질시스템 절차와 대조합니다.
- ISO 14971 위험도 계산과 IEC 62304 추적성 결과는 보조 지표이며 공식 판단을 대체하지 않습니다.
- RAG 근거는 원문, 버전, 작성일, 적용 범위를 직접 확인합니다.
- 환자정보, 개인정보, 영업비밀, 승인되지 않은 임상자료를 샘플이나 근거 문서로 사용하지 않습니다.

상세 기준은 [`ra-usage-precautions.md`](ra-usage-precautions.md)에 별도 정리했습니다.
