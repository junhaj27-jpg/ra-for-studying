# AIDLC-RA 사용 시 주의사항 및 RA 검토 기준

이 문서는 AIDLC-RA를 RA 취업 포트폴리오 또는 문서 자동화 보조 도구로 사용할 때 반드시 지켜야 할 주의사항을 정리합니다.

> 본 프로젝트는 의료 진단 또는 치료 목적의 소프트웨어가 아니라, 의료기기 개발 및 RA 문서 작성 과정을 보조하기 위한 포트폴리오용 문서 자동화 플랫폼입니다.

## 1. 사용 범위

AIDLC-RA는 다음 목적에 한해 사용합니다.

- 의료기기 개발 자료를 RA 문서 흐름으로 구조화
- 요구사항, 위험관리, 검증시험, 변경관리, 추적성 매트릭스 작성 보조
- 문서 간 누락, 불일치, 미검증 항목 검토 지원
- RAG 기반 근거 후보와 문서 출처 표시
- RA 실무 흐름 이해를 보여주는 포트폴리오 시연

AIDLC-RA는 다음 목적으로 사용하지 않습니다.

- 의료 진단
- 치료 결정
- 처방 추천
- 응급상황 판단
- 임상적 유효성 판단
- 실제 인허가 승인 여부 판단
- 법적 효력이 있는 규제 자문

## 2. 생성 문서 사용 시 주의사항

- 생성 결과는 초안이며 최종 승인 문서가 아닙니다.
- 실제 제출 전 문서번호, 버전, 승인권자, 서명, 개정이력, 회사 양식 반영이 필요합니다.
- 최신 법령, 고시, 가이드라인, 국제표준, 시험 기준, 기관별 양식과 반드시 대조해야 합니다.
- 같은 제품명이라도 사용 목적, 성능 주장, 대상 사용자, 사용 환경이 달라지면 문서 전체를 다시 검토해야 합니다.
- 자동 생성된 문구가 의료적 주장으로 읽힐 수 있는 경우 RA/QA 담당자가 직접 수정해야 합니다.

## 3. ISO 14971 위험관리 사용 시 주의사항

- 위험도 계산은 포트폴리오용 보조 로직이며 제조사의 공식 위험허용 기준표를 대체하지 않습니다.
- 발생가능성, 심각도, 위험수용 기준은 제품, 조직, 품질시스템 절차에 맞게 별도로 정의해야 합니다.
- 잔여위험 허용 여부는 단순 점수만으로 결정하지 않고, 위험통제 검증 결과와 이익-위험 판단을 함께 검토해야 합니다.
- 위험통제 항목은 반드시 검증시험 또는 문서 근거와 연결되어야 합니다.

## 4. IEC 62304 추적성 사용 시 주의사항

- 추적성 커버리지는 요구사항과 시험 연결 여부를 보여주는 보조 지표입니다.
- 소프트웨어 안전등급 결정, 적합성 판단, 시험 충분성 판단을 자동으로 대신하지 않습니다.
- 요구사항 ID, 위험 ID, 시험 ID, 변경 ID는 문서 전체에서 일관되게 유지해야 합니다.
- 요구사항 변경 시 SRS, 위험관리표, V&V 계획서, 통합시험 시나리오, 추적성 매트릭스를 함께 갱신해야 합니다.

## 5. RAG 근거 사용 시 주의사항

- RAG citation은 근거 후보를 표시하는 기능입니다.
- 사용자는 원문, 문서 버전, 작성일, 적용 범위, 최신성 여부를 직접 확인해야 합니다.
- 오래된 회의록, 잘못 업로드된 문서, 제품 범위와 맞지 않는 기준이 근거로 검색될 수 있습니다.
- 근거 문장과 생성 문서의 해석이 일치하는지 RA 담당자가 검토해야 합니다.

## 6. 변경관리 사용 시 주의사항

- 변경관리 영향 분석은 영향 가능성이 있는 문서를 추천하는 보조 기능입니다.
- 최종 재검증 필요 여부는 RA, QA, 개발, 시험 담당자가 함께 결정해야 합니다.
- 사용 목적, 성능 주장, 알고리즘, 데이터 처리, UI 문구, 경고문구 변경은 반드시 추적성 매트릭스와 V&V 계획에 반영해야 합니다.
- 변경 후 문서 간 ID, 용어, 합격 기준이 불일치하지 않는지 확인해야 합니다.

## 7. 데이터 보안 및 개인정보 주의사항

샘플 프로젝트나 RAG 근거에는 다음 정보를 넣지 않습니다.

- 환자정보
- 개인정보
- 영업비밀
- 승인되지 않은 임상자료
- 외부 반출이 제한된 회사 문서
- 실제 고객사 또는 병원 식별정보

포트폴리오 시연에는 익명화된 예시 데이터와 가상의 의료기기 시나리오를 사용합니다.

## 8. RA 검토 체크리스트

| 구분 | 확인 질문 |
| --- | --- |
| 사용 목적 | 진단, 치료, 처방, 응급 판단으로 오해될 표현이 없는가 |
| 요구사항 | 각 요구사항에 고유 ID와 검증 가능한 문장이 있는가 |
| 위험관리 | 위험통제 항목이 검증시험 또는 근거 문서와 연결되는가 |
| V&V | 요구사항과 위험통제를 확인할 시험 시나리오가 있는가 |
| 추적성 | 요구사항-위험-시험-결과-변경 ID가 하나의 흐름으로 연결되는가 |
| 변경관리 | 변경 영향 문서와 재검증 필요 여부가 기록되는가 |
| RAG 근거 | 출처, 버전, 작성일, 적용 범위가 확인되었는가 |
| 출력 문서 | 문서번호, 버전, 승인자, 개정이력, 서명란이 필요한가 |
| 데이터 보안 | 환자정보, 개인정보, 영업비밀이 포함되지 않았는가 |

## 9. 포트폴리오 설명 문장

면접 또는 README에서 다음처럼 설명할 수 있습니다.

> AIDLC-RA는 실제 인허가 판단을 대신하는 도구가 아니라, 의료기기 개발 과정에서 요구사항, 위험관리, 검증시험, 변경관리, 추적성 문서를 일관되게 작성하도록 돕는 RA 문서 자동화 포트폴리오입니다. 사용 시 주의사항과 RA 검토 체크리스트를 별도로 정리하여 생성형 AI 결과를 그대로 사용하지 않고, 전문가 검토와 문서 근거 확인이 필요하다는 점을 명확히 했습니다.
